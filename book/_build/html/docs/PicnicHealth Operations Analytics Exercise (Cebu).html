Backlogs¶

Task:¶

  • Analyze the current backlog of work

Assumptions:¶

  • Defined Backlogs as tasks without completed_at date

  • Created a task_group variable for the task_types based on their common context

  • Grouped backlogs into two groups: stand-alone tasks and transition tasks

Backlogs¶

Analysis:¶

  • Filtered tasks without completed_at date

  • Created a task_group variable that groups major tasks according to common context

  • Created a task_transistion variable that indicates previous to current task transition

  • Grouped tasks according to previous task: stand-alone (NA previous) and transition tasks (other task previous)

  • Counted the number of tasks per task_type,task_group, and previous_task_type

  • Filtered count of tasks above 250 to look at major groups

Backlogs¶

Findings:¶

  • There are a total of 8,427 backlog tasks

  • Count of backlog tasks range from 1 to 1042 according to the grouping

  • Stand-alone tasks mainly come from the MISC and PROCESS task groups

  • The majority of stand-alone tasks are the MESSAGE_USER_FOR_ONBOARDING_INFO and QUALITY_CHECK_SEGMENTATION

  • Transition tasks mainly come from the MESSAGE and PROCESS task groups

  • Most backlogs transistion from OCR tasks to PROCESS TASKS

  • Specifically, CORRECT_EXCERPT_OCR to PROCESS_EXCERPTS take up majority of the transition backlogs

Turnaround¶

Task:¶

  • Analyze the turnaround times by task and for the entire production process

Assumptions:¶

  • Defined Turnaround as the time duration it takes to finish a task

  • Time duration is difference between completed_at and created_at

  • Defined a task_group for the task_types based on their common context

Turnaround¶

Analysis:¶

  • Information from records were joined to tasks based on task_type to match record into to task

  • Filtered tasks with completed_at date

  • Calculated task_duration in seconds and in hours

  • Added a stand-alone boolean variable to check if transition task

  • Calculated average and count-weighted average task_duration_hours

  • Plotted the task duration distribution per task_type and task_group

  • Investigated transistion/stand-alone tasks by task_group

  • Investigated if number of pages in record affect duration

Turnaround¶

Findings:¶

  • Count-weighting highlights the significance of high-frequency tasks compared to normal averaging

  • Specifically, CALL tasks are insignificant despite longer turnaround due to lower frequency

  • Both averaging calculations show that PROCESS and QUALITY task groups have significant turnaround times

  • The distribution of task duration for task_types are usually unimodal but some have multiple modes

  • This indicate that there might be a factor that affects the duration that splits the distribution

  • No apparent difference between Stand-alone and Transition task but looking at task_group level shows difference

  • Specifically, QUALITY tasks have higher turnaround if Stand-alone compared to Transition

  • Number of pages in record does not have a strong relationship between task duration
